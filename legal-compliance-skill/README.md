# Legal Compliance Skill (1688 法务合规审核助手)

## 📖 项目简介
本项目是专为 **电商平台法务部** 定制的 AI 审核技能（Skill）。它基于内部《素材审核逻辑整理》文档构建，旨在自动化审阅业务部门对外发布的宣传素材（包括文字、图片、视频、PPT、海报等），确保所有内容符合阿里巴巴集团及 1688 平台的合规要求。

## 核心功能
1. **全素材类型覆盖**：支持文本、图片描述/分析、视频脚本/时间轴、PPT 内容等多模态审核。
2. **严格规则遵循**：内置详细的数据披露、版权图片、Logo 使用、绝对化用语、跨境业务限制等合规逻辑。
3. **智能交互机制**：
   - 若遇到文档未涵盖的素材类型或场景，自动暂停并主动向用户发问，请求指示。
   - 对模糊地带提供风险提示并要求核实真实性。
4. **专业化输出报告**：
   - **文字/PPT**：生成标准的 **Redline (修订模式)** 对比，清晰展示删除与新增内容。
   - **视频**：精确指出 **几分几秒** 的违规点，并提供修改建议。
   - **图片**：若原图违规，自动生成符合合规要求的 **AI 绘画提示词 (Prompt)**，供设计师重新制作无风险素材。

## 目录结构
- `skill_definition.yaml`: 核心系统指令，定义 AI 的角色、工作流和输出格式。
- `knowledge_base/`: 
  - `compliance_rules.md`: 完整的合规规则知识库（源自内部文档）。
  - `forbidden_keywords.json`: 结构化禁用词表（绝对化用语、敏感词等）。
- `templates/`: 标准化输出模板，确保报告格式统一。
- `examples/`: 典型违规案例库，用于 Few-Shot 学习，提升 AI 判断准确度。

## 如何使用
### 前置准备
1. 将此仓库克隆或上传至您的 GitHub 空间。
2. 在您的 AI Agent 平台（如 GitHub Copilot Workspace, LangChain, Dify 等）中配置：
   - **System Prompt**: 读取 `skill_definition.yaml` 的内容。
   - **Knowledge Base**: 索引 `knowledge_base/` 文件夹下的所有文件。

### 使用场景示例
#### 场景 A：审核宣传文案
> **用户输入**: "这是我们要发的公众号文章草稿：1688 全网销量第一，保证商家无忧入驻，绝对是最好的 B2B 平台！"
> **AI 响应**: 
> - 识别绝对化用语 ("第一", "最好", "绝对", "保证")。
> - 输出 Redline 修改建议，将违禁词替换为合规表述。

#### 场景 B：审核视频脚本
> **用户输入**: "视频脚本：00:15 展示竞品 Logo 进行对比；00:30 主播说'这就是山姆平替'。"
> **AI 响应**:
> - 指出 00:15 违反第三方 Logo 使用规范。
> - 指出 00:30 违反北美/通用不正当竞争规范（"山姆平替"）。
> - 给出具体时间点修改方案。

#### 场景 C：审核设计图概念
> **用户输入**: "设计图里想用 Loopy 的形象来做春节海报，吸引年轻商家。"
> **AI 响应**:
> - 警告：知名卡通形象（Loopy）存在高侵权风险，禁止使用。
> - 行动：生成一段安全的 Midjourney/Stable Diffusion 提示词，描述一个原创的、喜庆的、无版权风险的春节商务形象。
